# 分享缩略图修复测试指南
> 日期：2026-03-30

## 修复内容概述

修复了以下页面的分享缩略图问题：
1. **详情页** - 分享特产时不显示产品缩略图
2. **首页** - 分享小程序时不显示热门特产图片
3. **订单详情页** - 分享订单时不显示相关产品图片
4. **朋友圈分享** - 分享到朋友圈不显示缩略图

## 问题原因

**根本原因**：微信分享要求 `imageUrl` 必须是 HTTPS 链接，不能是 `cloud://` 格式的云存储 fileID。

**修复方案**：在分享函数中将 `cloud://` 格式的图片转换为 HTTPS 临时链接。

## 技术实现验证

### 1. 图片链接转换验证

**修复代码位置**：
- `pages/detail/index.js` - `onShareAppMessage` 和 `onShareTimeline`
- `pages/index/index.js` - `onShareAppMessage` 和 `onShareTimeline`
- `pages/order-detail/index.js` - `onShareAppMessage` 和 `onShareTimeline`

**转换逻辑**：
```javascript
// 如果是 cloud:// 格式，转换为 HTTPS 临时链接
if (imageUrl && imageUrl.startsWith('cloud://')) {
  try {
    const tempRes = await wx.cloud.getTempFileURL({
      fileList: [imageUrl]
    })
    if (tempRes.fileList && tempRes.fileList[0]) {
      imageUrl = tempRes.fileList[0].tempFileURL
    }
  } catch (e) {
    console.error('转换图片链接失败:', e)
    // 使用默认图片
  }
}
```

## 常见问题排查

### Q1: 分享卡片仍然不显示图片
**可能原因**：
1. 图片链接转换失败
2. 图片链接不是 HTTPS 格式
3. 图片链接已过期

**解决方案**：
1. 检查控制台是否有 `getTempFileURL` 错误
2. 确认转换后的链接是 HTTPS 格式
3. 临时链接有效期2小时，需要重新获取

### Q2: 朋友圈分享不显示缩略图
**可能原因**：
1. `onShareTimeline` 没有返回 `imageUrl` 字段
2. `imageUrl` 不是 HTTPS 链接
3. 图片尺寸不符合要求

**解决方案**：
1. 确保 `onShareTimeline` 返回 `imageUrl` 字段
2. 确保图片链接是 HTTPS 格式
3. 建议图片尺寸 800×640（5:4比例）

### Q3: 分享功能报错
**错误信息**：`getTempFileURL:fail ...`

**可能原因**：
1. `cloud://` 格式不正确
2. 文件不存在
3. 权限问题

**解决方案**：
1. 检查 `cloud://` 格式是否正确
2. 确认文件在云存储中存在
3. 检查云存储权限设置
