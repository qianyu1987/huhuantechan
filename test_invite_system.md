- 测试时间: 2026-03-30

## 测试用例

### 测试用例1：邀请码生成和分享
**步骤**：
1. 用户A登录小程序
2. 进入"邀请好友"页面
3. 检查是否生成邀请码
4. 检查是否生成小程序码
5. 测试复制邀请码功能
6. 测试分享给好友功能

**预期结果**：
- 用户A获得唯一的邀请码（格式：INV + openid后6位）
- 成功生成小程序码
- 可以复制邀请码
- 可以分享给好友

### 测试用例2：邀请关系绑定
**步骤**：
1. 用户B通过用户A的邀请链接打开小程序
2. 用户B完成注册
3. 检查邀请关系是否绑定
4. 检查注册奖励是否发放

**预期结果**：
- 用户B的`invitedBy`字段设置为用户A的openid
- 用户A获得10积分奖励
- 用户B获得5积分奖励
- 在`points_log`和`invite_rewards`中有相应记录

### 测试用例3：首次互换奖励
**步骤**：
1. 用户B（被邀请人）发布一个特产
2. 用户C（其他用户）与用户B完成互换
3. 检查首次互换奖励是否发放

**预期结果**：
- 用户B获得20积分首次互换奖励
- 用户A（邀请人）获得20积分好友首次互换奖励
- 在`points_log`和`invite_rewards`中有相应记录

### 测试用例4：防作弊机制
**步骤**：
1. 用户A邀请用户B（正常）
2. 用户A尝试邀请用户C（第11次邀请）
3. 用户B尝试再次绑定其他邀请关系
4. 用户A尝试邀请自己

**预期结果**：
- 用户A第1-10次邀请成功
- 用户A第11次邀请失败，提示"今日邀请次数已达上限"
- 用户B再次绑定失败，提示"已绑定过邀请关系"
- 用户A邀请自己失败，提示"不能邀请自己"

## 预期问题及解决方案

### 可能的问题
1. **邀请码重复** - 使用openid后6位确保唯一性
2. **奖励重复发放** - 使用`firstSwapRewardSent`标志防止重复
3. **并发问题** - 使用数据库事务或乐观锁
4. **网络超时** - 添加重试机制

### 解决方案
1. 邀请码生成：`INV + openid.slice(-6).toUpperCase()`
2. 奖励防重：检查`firstSwapRewardSent`字段
3. 并发控制：使用`_id`精准更新
4. 错误处理：try-catch包裹关键操作
