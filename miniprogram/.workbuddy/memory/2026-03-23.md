## 安卓手机号验证修复（晚上）

### 问题描述
用户在安卓手机上点击"快速验证手机号"按钮时，系统调用 `getPhoneNumber` API 失败，提示"兼容模式调用失败：推送授权页失败"。

### 根本原因
- 安卓平台上，如果用户未完成微信登录（没有获取到 openid/session_key），直接调用 `getPhoneNumber` 会失败
- 原来的 `phone-verify` 组件没有检查用户是否已登录

### 解决方案
1. **phone-verify 组件增强登录状态检查**:
   - 新增 `isLoggedIn` 数据字段
   - 新增 `checkLoginStatus()` 方法检查 `app.globalData.openid`
   - 未登录时显示"先登录微信"按钮，点击弹窗提示用户先点击头像授权
   - 已登录时才显示手机号验证按钮

2. **mine 页面登录回调更新**:
   - 给 `phone-verify` 组件添加 `id="phoneVerify"`
   - 在 `saveUserProfile()` 成功后调用 `phoneVerify.checkLoginStatus()` 更新组件状态

3. **云函数 userInit 增强**:
   - `updateProfile` action 返回 `openid`，供前端判断登录状态

### 修改文件
- `components/phone-verify/index.js` - 登录状态检查逻辑

## 多端开发数据同步问题修复（之前完成）

### 问题描述
- 小程序上的数据不会同步到多端开发环境(HarmonyOS)
- 根本原因: 微信小程序和 HarmonyOS 使用不同的用户标识(openid),导致无法识别为同一用户

### 解决方案
使用 **UnionID** 实现跨平台用户统一识别

#### 修改内容
1. **云函数 userInit/index.js**:
   - 新增 `unionid` 字段存储微信 UnionID
   - 优先使用 UNIONID 查询用户,确保跨平台识别
   - 在用户初始化时自动存储 UNIONID
   - 已有用户首次登录时自动补充 UNIONID

2. **查询优先级**:
   ```
   1. UNIONID 查询 (跨平台统一标识)
   2. _openid 查询 (云函数自动注入)
   3. openid 字段查询 (兼容旧数据)
   ```

3. **修改的 action**:
   - `init`: 初始化用户时存储 UNIONID
   - `updateProfile`: 更新资料时优先用 UNIONID 查询
   - `saveProfile`: 保存资料时优先用 UNIONID 查询
   - `getMyProfile`: 获取资料时优先用 UNIONID 查询

### UnionID 机制说明
- UnionID 是微信开放平台提供的统一用户标识
- 同一微信开放平台下的所有应用(小程序、公众号、APP)共享相同的 UnionID
- OpenID 在不同应用中不同,但 UnionID 相同
- 前提条件: 需要将小程序绑定到微信开放平台账号

### 部署步骤
1. **部署云函数**:
   - 右键 `cloudfunctions/userInit` → "上传并部署: 云端安装依赖"
   
2. **验证部署**:
   - 查看云函数日志,应显示 `[userInit] 平台信息: { openid, unionid, ... }`
   - 数据库 `users` 集合应包含 `unionid` 字段

3. **测试跨平台**:
   - 在微信小程序端登录,查看是否有 unionid
   - 在 HarmonyOS 端登录,应该能找到相同用户记录
   - 检查数据是否同步

### 注意事项
- 如果用户未绑定微信开放平台,unionid 可能为空,会降级使用 openid
- 旧用户首次登录会自动补充 unionid 字段
- 新用户创建时会自动存储 unionid

### 后续优化建议
1. 添加数据迁移脚本,为所有旧用户补充 unionid
2. 在前端显示"跨平台同步"状态提示
3. 添加账号绑定/解绑功能
