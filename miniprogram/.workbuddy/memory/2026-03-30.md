## 管理后台等级管理「调等级」「录入/修改押金」弹窗不显示 Bug 修复（13:28）

### 问题
管理后台等级管理 Tab，点击「调等级」和「录入/修改押金」按钮后弹窗不弹出，完全无响应。

### 根本原因
**微信小程序 dataset 传递复杂对象被截断**：WXML 中通过 `data-user="{{item}}"` 把整个用户对象传给事件处理方法，当对象包含嵌套字段（daigouStats/daigouVerify 等）时，微信小程序对 dataset 的大小有限制，超大对象会被截断或丢弃，导致 `e.currentTarget.dataset.user` 为 `undefined`。弹窗内 `wx:if="{{levelModalUser}}"` / `wx:if="{{depositInputUser}}"` 为 false，内容区不显示，弹窗黑色背景可能一闪而过，看起来就是"点击无效"。

**同时**：调等级弹窗和录入押金弹窗的 `modal-overlay` 缺少 `position: fixed` 内联样式（用户编辑弹窗有而这两个没有），CSS class 在某些手机上可能不生效。

### 修复
1. **WXML**：`data-user="{{item}}"` → `data-openid="{{item.openid}}"` —— 只传 openid（字符串），彻底避免 dataset 截断问题
2. **JS**：`openLevelModal` / `openDepositInputModal` 方法改为通过 openid 从 `this.data.levelUsers` 数组中 `find()` 查找完整用户对象
3. **WXML 弹窗**：给两个弹窗的 `modal-overlay` 和 `modal-box` 加上与用户编辑弹窗一致的 `position: fixed` 内联样式，确保在任何手机上都能正确覆盖全屏

### 改动文件
- `miniprogram/pages/admin/index.wxml`：dataset 改 openid，弹窗加 fixed 样式
- `miniprogram/pages/admin/index.js`：openLevelModal / openDepositInputModal 逻辑修正

## 用户详情页（user-profile）空白问题修复（15:21）

### 场景
从特产详情页点击发布者头像/名字，跳转到 user-profile 页面后内容不显示（空白）。

### 根本原因（共3处）

1. **`productMgr` `detail` action — publisher.openid 赋值优先级错误（第 900 行）**
   - 原来：`publisher.openid = publisher._openid || product.data.openid || publisherOpenid`
   - `.field()` 投影不能获取系统字段 `_openid`（总是 undefined），如果 products 也没有显式 `openid` 字段，则传给前端的 openid 为 undefined
   - 修复：改为 `publisher.openid = publisherOpenid || publisher._openid || product.data.openid`，优先用从 doc.get() 直接读到的 `publisherOpenid`（系统字段通过 doc 可以读到）

## 提现申请页面数字不显示问题修复（19:55）

### 问题
用户反馈提现申请页面中的数字不显示

### 原因分析
1. **数据类型问题**：云函数返回的金额数据可能不是数字类型（可能是字符串或undefined）
2. **toFixed调用错误**：WXML中直接调用`toFixed(2)`方法，如果数据不是数字会报错
3. **数据加载问题**：前端加载数据时未确保所有金额字段都是数字类型

### 修复方案
1. **添加formatAmount辅助函数**：
   ```javascript
   function formatAmount(value) {
     const num = parseFloat(value)
     return isNaN(num) ? '0.00' : num.toFixed(2)
   }
   ```

2. **修改数据加载逻辑**：
   - 在`loadWithdrawalConfig`函数中添加类型转换，确保`withdrawalThreshold`、`walletBalance`、`availableAmount`、`maxWithdrawalAmount`、`withdrawalFeeRate`都是数字类型
   - 使用`parseFloat()`转换所有金额字段

3. **修改WXML显示**：
   - 将所有`toFixed(2)`调用改为`formatAmount()`函数调用
   - 确保金额显示安全，即使数据有问题也能显示默认值

4. **修复验证逻辑**：
   - 在`validateForm`和`submitWithdrawal`函数中修复`toFixed(2)`调用，确保参数是数字
   - 添加安全调用：`(withdrawalThreshold || 0).toFixed(2)`

### 修改文件
- `pages/withdrawal/index.js`：添加类型转换和`formatAmount`函数
- `pages/withdrawal/index.wxml`：将`toFixed(2)`调用改为`formatAmount`函数调用

### 测试验证
1. 创建测试脚本验证修复效果
2. `formatAmount`函数能正确处理各种输入（undefined、null、字符串、数字）
3. 数据加载逻辑能正确转换字符串数字为数字类型
4. 错误数据会被安全地处理为默认值
