P0 核心功能（本周完成 - 15h）

订单超时机制

▼

功能说明

  • 确认后 72小时 未发货，自动取消订单
  • 订单取消后，双方特产恢复为 active 状态
  • 发送系统通知给双方
  • 记录取消原因和时间

业务规则

场景触发条件处理方式
超时未发货确认后 72h自动取消 + 扣信用分
超时未收货发货后 7 天自动确认收货

技术方案

方案：云函数定时触发器

每小时执行一次，批量处理超时订单

// checkOrderTimeout 云函数 const now = new Date() const deadline = new Date(now - 72*60*60*1000) const orders = await db.collection('orders') .where({ status: 'confirmed', confirmTime: db.command.lt(deadline) }).get() // 批量取消订单 for (const order of orders.data) { await cancelOrder(order._id, '超时未发货') }

// orders 表新增字段 { confirmTime: Date, // 确认时间 shipDeadline: Date, // 发货截止时间 autoCancelReason: String }
预估工作量：4h （云函数 2h + 定时器 0.5h + 取消逻辑 1h + 前端 0.5h）

P3 后期规划（42h+）

会员订阅

▼
不建议现在做付费会员
  • 需要微信支付资质，申请周期长
  • 功能复杂（42h+），ROI 不确定
  • 用户量不够，付费转化率可能很低

建议替代方案：积分会员

权益普通用户会员用户复杂度
发布特产数量5 个20 个低
神秘特产发布1 个/月无限制低
每日签到积分+1+2低
优先匹配普通优先展示中
专属标识无会员徽章低
积分会员工作量：20h（无支付集成）

风险与建议

风险点

  • 消息推送：需申请模板消息，审核可能不通过
  • 会员订阅：需微信支付资质，申请周期长
  • 防刷机制：邀请奖励可能被恶意刷取

建议

  • 先做低风险功能：订单超时、信用分、首次互换奖励
  • 同步申请资质：消息模板、微信支付
  • 灰度发布：新功能先小范围测试
  • 数据驱动：积累数据后再做付费会员
