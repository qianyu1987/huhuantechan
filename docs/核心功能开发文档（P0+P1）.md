## 一、积分充值（微信支付）

### 1.1 功能概述

用户在小程序内通过微信支付购买积分，积分可用于：
- 解锁平台高级功能（优先展示位、加急互换等，未来迭代）
- 代购支付
- 抵扣平台服务费

**核心变现入口，优先开发。**

### 1.2 业务规则

| 充值档位 | 金额（元） | 获得积分 | 赠送积分 |
|---------|-----------|---------|---------|
| 小试牛刀 | 1 | 100 | — |
| 基础包 | 6 | 660 | +60（10%） |
| 进阶包 | 18 | 2000 | +200（11%） |
| 超值包 | 48 | 5500 | +500（10%） |
| 土豪包 | 98 | 12000 | +2000（20%） |

- 积分不可提现，不可转让
- 积分永久有效（未来可设置过期策略）
- 消费积分时按获取时间先进先出（FIFO）

### 1.3 数据库变更

#### 现有字段复用
`users` 集合已有 `points` 字段，**无需新增字段**。

### 1.4 微信支付接入流程

```
小程序端                    paymentMgr云函数                微信支付平台
   │                              │                              │
   │── wx.requestPayment ────────►│                              │
   │   (前端调起支付)              │── POST /v3/pay/transactions/jsapi ──►│
   │                              │◄── prepay_id ─────────────────│
   │◄── 返回支付参数 ──────────────│                              │
   │                              │                              │
   │── 用户完成支付 ──────────────►│                              │
   │                              │◄── 支付回调 notify_url ────────│
   │                              │  (验签 → 入账积分 → 更新订单)   │
   │◄── 支付成功通知 ──────────────│                              │
```

**关键安全要求**：
1. 签名验证必须在云函数服务端完成，前端不得处理
2. 回调接口需幂等处理（同一笔订单重复回调只入账一次）
3. 使用微信支付 v3 API（RSA + AES-256-GCM）

## 二、纠纷仲裁（人工介入）

### 2.1 功能概述

互换双方发生争议时，任一方可申请平台介入仲裁。管理员在后台查看证据、作出裁决，系统自动执行处罚/补偿。

**核心价值**：建立平台信任，降低用户损失风险。

### 2.2 业务流程

```
申请仲裁                 管理员受理                  作出裁决
    │                        │                           │
用户发起申请              收到系统通知                决定哪方胜出
填写纠纷描述              查看双方证据               执行自动处理：
上传凭证（图片）          联系双方了解情况           • 退回/补偿积分
                         可设置"处理中"状态          • 信用分扣减
                                                    • 关闭/回滚订单
```

### 2.3 纠纷状态机

```
pending（待受理）
  └──► processing（处理中，管理员已介入）
         ├──► resolved_buyer_win（买方胜）
         ├──► resolved_seller_win（卖方胜）
         └──► resolved_split（各退一步，双方平局）
```

## 三、订单状态服务通知推送

### 3.1 功能概述

在订单各关键节点，通过**微信服务消息**（subscribeMessage）向相关用户推送通知，提升互动效率和用户体验。

### 3.2 通知场景矩阵

| 触发事件 | 推送对象 | 消息模板关键词 |
|---------|---------|--------------|
| 对方申请互换 | 商品主人 | 申请人昵称、商品名、申请时间 |
| 互换申请被接受 | 申请人 | 对方昵称、商品名、需发货提醒 |
| 互换申请被拒绝 | 申请人 | 对方昵称、商品名 |
| 对方已发货 | 收货方 | 快递公司、快递单号、发货时间 |
| 对方已确认收货 | 发货方 | 提示可评价 |
| 新订单被申请（代购） | 卖家 | 买家昵称、商品、金额 |
| 申请仲裁 | 被申请方 | 纠纷类型、订单号 |
| 仲裁结果 | 双方 | 处理结果说明 |

### 3.3 技术方案

微信小程序服务消息需用户**主动订阅**（`wx.requestSubscribeMessage`），并且：
- 用户订阅后，消息只能发送**一次**（一次性订阅）
- 或申请**长期订阅**（需微信审核，适用于交易类场景）

**推荐方案：触发式订阅**  
在关键操作前弹出订阅授权，例如：
- 点击"申请互换"前 → 订阅"互换申请结果通知"
- 点击"确认发货"前 → 订阅"对方确认收货通知"

### 3.4 数据库变更

## 四、订单超时机制

### 4.1 功能概述

互换订单确认后若长时间无发货，系统自动取消订单并释放商品，防止资源长期锁定。

### 4.2 超时规则

| 状态 | 超时时间 | 触发动作 |
|------|---------|---------|
| `confirmed`（已确认，等待发货） | 72小时 | 自动取消订单，信用分 -3 |
| `shipped`（已发货，等待收货） | 14天 | 自动确认收货（友好机制） |
| `pending`（待接受申请） | 72小时 | 自动关闭申请（轻量处理，不扣分） |

### 4.3 实现方案：定时云函数

**在现有 `dailyTasks` 云函数中新增超时检查逻辑**（不新建云函数）：

```js
// dailyTasks/index.js 新增 action: checkOrderTimeout
async function checkOrderTimeout() {
  const now = new Date()
  const db = cloud.database()
  const _ = db.command

  // 1. 处理 confirmed 超时（72h未发货 → 取消）
  const confirmedTimeout = new Date(now - 72 * 60 * 60 * 1000)
  const confirmedOrders = await db.collection('swapOrders')
    .where({
      status: 'confirmed',
      confirmTime: _.lt(confirmedTimeout)
    })
    .get()

  for (const order of confirmedOrders.data) {
    // 取消订单
    await db.collection('swapOrders').doc(order._id).update({
      data: {
        status: 'cancelled',
        cancelReason: '超时未发货，系统自动取消',
        updateTime: db.serverDate()
      }
    })

    // 释放商品（将 in_swap 状态改回 active）
    if (order.offerProductId) {
      await db.collection('products').doc(order.offerProductId).update({
        data: { status: 'active' }
      })
    }
    if (order.receiveProductId) {
      await db.collection('products').doc(order.receiveProductId).update({
        data: { status: 'active' }
      })
    }

    // 扣除未发货方信用分
    const timeoutOpenid = /* 判断哪方未发货 */ order.offerOpenid
    await db.collection('users').where({ _openid: timeoutOpenid }).update({
      data: { creditScore: _.inc(-3) }
    })

    // 记录信用分日志
    await db.collection('credit_logs').add({
      data: {
        openid: timeoutOpenid,
        delta: -3,
        reason: '订单超时未发货（系统自动）',
        relatedId: order._id,
        createTime: db.serverDate()
      }
    })

    // 发服务通知（如有订阅）
    // await notifyHelper.send(...)
  }

  // 2. 处理 shipped 超时（14天未确认 → 自动确认）
  const shippedTimeout = new Date(now - 14 * 24 * 60 * 60 * 1000)
  const shippedOrders = await db.collection('swapOrders')
    .where({
      status: 'shipped',
      shipTime: _.lt(shippedTimeout)
    })
    .get()

  for (const order of shippedOrders.data) {
    await db.collection('swapOrders').doc(order._id).update({
      data: {
        status: 'completed',
        receiveTime: db.serverDate(),
        autoReceived: true,
        updateTime: db.serverDate()
      }
    })
    // 加信用分 +5
    // 更新 swapCount
  }

  return {
    cancelledCount: confirmedOrders.data.length,
    autoReceivedCount: shippedOrders.data.length
  }
}
```

## 五、邀请好友（裂变）

### 5.1 功能概述

用户通过专属邀请链接/小程序码邀请新用户注册，双方均获得积分奖励，驱动裂变增长。

### 7.5 云函数设计

**封装公共函数 `updateCreditScore`**（在 `orderMgr`/`adminMgr` 复用）：

```js
// cloudfunctions/common/creditHelper.js
async function updateCreditScore(db, openid, delta, reason, type, relatedId) {
  const user = await db.collection('users')
    .where({ _openid: openid }).field({ creditScore: true, _id: true }).get()
  
  if (!user.data.length) return false
  
  const currentScore = user.data[0].creditScore || 100
  const newScore = Math.max(currentScore + delta, -999) // 最低 -999
  
  // 更新信用分
  await db.collection('users').doc(user.data[0]._id).update({
    data: {
      creditScore: newScore,
      creditLevel: calcCreditLevel(newScore),
      lastCreditUpdate: db.serverDate()
    }
  })
  
  // 写日志
  await db.collection('credit_logs').add({
    data: { openid, delta, reason, type, relatedId, newScore, createTime: db.serverDate() }
  })
  
  // 封号检测
  if (newScore < 0) {
    await db.collection('users').doc(user.data[0]._id).update({
      data: { status: 'banned', banReason: '信用分耗尽' }
    })
  }
  
  return { newScore, creditLevel: calcCreditLevel(newScore) }
}

function calcCreditLevel(score) {
  if (score >= 150) return 'S'
  if (score >= 120) return 'A'
  if (score >= 90)  return 'B'
  if (score >= 60)  return 'C'
  if (score >= 30)  return 'D'
  if (score >= 0)   return 'E'
  return 'banned'
}
```
