# 安卓端网络问题解决方案

## 问题描述
安卓端打开小程序后无数据显示,提示"网络错误",无法连接云开发。

## 已修复问题

### 1. urlCheck 配置 (✅ 已修复)
**问题**: `project.config.json` 中 `urlCheck: true` 导致安卓端无法发起云开发请求。

**原因**: 
- `urlCheck: true` 会严格检查请求域名合法性
- 云开发使用特殊的 `cloud://` 协议,不是常规的 https 域名
- 安卓端对域名校验比iOS更严格

**修复**:
```json
// project.config.json
{
  "setting": {
    "urlCheck": false
  }
}
```

### 2. 网络诊断工具 (✅ 已添加)
新增网络诊断页面 `pages/network-test/`,用于排查网络问题:
- 检测云开发SDK加载状态
- 测试云函数连通性
- 测试数据库查询
- 测试云存储访问
- 测试普通网络请求

**访问方式**: 我的 → 设置 → 网络诊断

**云开发域名** (自动配置,无需手动添加):

- 如果仍有问题,检查云开发控制台的网络设置

### 2. 安卓端权限配置
如果是打包为Android App (多端应用),需要检查:

**AndroidManifest.xml** (如果有):
```xml
<uses-permission android:name="android.permission.INTERNET"/>
<uses-permission android:name="android.permission.ACCESS_NETWORK_STATE"/>
```

**网络安全配置** `res/xml/network_security_config.xml`:
```xml
<?xml version="1.0" encoding="utf-8"?>
<network-security-config>
    <base-config cleartextTrafficPermitted="true">
        <trust-anchors>
            <certificates src="system" />
        </trust-anchors>
    </base-config>
</network-security-config>
```

### 3. 云开发环境配置
检查云开发控制台:
1. 确认环境ID: `cloud1-3g4sjhqr5e28e54e`
2. 检查环境状态是否正常
3. 查看云函数日志是否有错误
4. 检查数据库访问权限设置

## 常见错误码

### -601002
**含义**: 云函数环境错误  
**解决**: 
- 重试机制已内置(最多3次)
- 检查云开发环境状态
- 确认云函数已部署

### -1
**含义**: 网络错误或超时  
**解决**:
- 检查网络连接
- 检查域名配置
- 检查防火墙设置

### 404 / 403
**含义**: 域名校验失败  
**解决**:
- 确认 `urlCheck: false`
- 检查小程序后台域名配置
- 检查云开发环境权限
