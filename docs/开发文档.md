## 一、项目概述

**换特产** 是一款基于微信小程序的特产互换社交平台，让用户可以发布自己家乡的特产，与全国各地的用户进行等价物物交换。

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端框架 | 原生微信小程序（无第三方框架） |
| UI | 自定义组件 + WeUI 字体 |
| 后端 | 微信云开发（CloudBase） |
| 数据库 | 云开发数据库（文档型 NoSQL） |
| 文件存储 | 微信云存储（cloud:// 协议） |
| 云函数运行时 | Node.js + wx-server-sdk |
| 云环境 ID | cloud1-3g4sjhqr5e28e54e |

### 项目结构

```
specialty-swap/
├── miniprogram/               # 小程序前端
│   ├── app.js                 # 全局逻辑（初始化、登录）
│   ├── app.json               # 页面路由、TabBar、权限配置
│   ├── app.wxss               # 全局样式变量（CSS Variables）
│   ├── pages/                 # 22个页面
│   ├── components/            # 公共组件（tab-bar 等）
│   ├── utils/                 # 工具函数和常量
│   └── images/                # 静态资源
├── cloudfunctions/            # 云函数
│   ├── userInit/              # 用户管理
│   ├── productMgr/            # 特产管理
│   ├── orderMgr/              # 订单管理
│   ├── reviewMgr/             # 评价管理
│   ├── adminMgr/              # 后台管理
│   └── testConnect/           # 连通性测试
└── docs/                      # 文档
```

## 二、功能模块清单

### 2.1 用户体系

| 功能 | 说明 |
|------|------|
| 微信登录 | 自动获取 openid，首次登录创建用户记录 |
| 个人资料 | 头像、昵称、家乡省份（家乡不可修改） |
| 信用分 | 初始 100 分，4 级信用等级，影响推荐排序 |
| 省份集章 | 每完成一次跨省互换自动收集对方省份徽章 |
| 邀请系统 | 生成邀请码，被邀请人绑定后双方获得积分 |
| 收货地址 | 增删改查，支持设置默认地址 |

### 2.2 特产管理

| 功能 | 说明 |
|------|------|
| 发布特产 | 上传图片(最多6张)、选品类/价值/描述标签/想换地区 |
| 神秘特产 | 盲盒模式，发布时隐藏详情，只显示省份 |
| 编辑特产 | 修改已发布的特产信息 |
| 下架特产 | 将特产从展示列表移除 |
| 搜索 | 关键词搜索，本地历史记录 |
| 收藏 | 收藏/取消收藏特产 |
| 浏览量 | 每次查看详情自动+1 |

### 2.3 互换流程

| 功能 | 说明 |
|------|------|
| 智能匹配 | 基于想换地区、品类、价值区间推荐 |
| 发起互换 | 选择自己特产 → 选择对方特产 → 创建订单 |
| 订单管理 | 全部/待确认/进行中/已完成 四个Tab |
| 物流填写 | 快递公司+运单号 |
| 确认收货 | 双方确认后订单完成 |
| 纠纷处理 | 可申请纠纷，管理员介入 |

### 2.4 评价体系

| 功能 | 说明 |
|------|------|
| 评价 | 订单完成后可评价，好评/差评 + 8种标签 |
| 信用联动 | 好评 +2 分，差评 -10 分 |
| 评价展示 | 用户主页展示收到的评价 |

### 2.5 特色功能

| 功能 | 说明 |
|------|------|
| 神秘特产（盲盒） | 只展示省份和emoji，收货后揭晓内容 |
| 省份集章 | 34省份集章墙，激励跨省互换 |
| AI 客服 | 内置35条知识库，关键词匹配智能回答 |
| 瀑布流布局 | 首页双列瀑布流展示，适配不同屏幕 |
| 管理后台 | 数据统计、用户管理、内容审核 |

## 七、信用与积分系统

### 信用分规则

- 初始分：100
- 范围：0-100
- 好评加分：+2
- 差评扣分：-10
- 互换完成：+5
- 确认后取消：-5
- 纠纷败诉：-15
- 信用分影响推荐排序权重

### 积分系统

- 邀请好友注册：双方各得积分
- 完成互换：获得积分
- 日常签到：获得积分
- 积分用途：当前版本仅展示，后续可扩展为兑换、抵扣

## 八、图片处理流程

```
用户选图 → wx.chooseMedia()
    ↓
压缩处理 → wx.compressImage() (quality: 80)
    ↓
上传云存 → wx.cloud.uploadFile() → cloud://fileID
    ↓
前端展示 → processImageUrl() → wx.cloud.getTempFileURL() → https:// 临时链接
    ↓
跨用户展示 → 云函数 resolveCloudUrl() 批量转换
```

- 每次最多上传 6 张
- 批量转换每批最多 50 个 fileID
- 临时链接有效期约 2 小时

## 九、已知问题与修复记录

| 问题 | 原因 | 修复 |
|------|------|------|
| 包体积超 1.5MB | tdesign 库完全未使用但被打包 | 删除 tdesign，替换为原生组件 |
| 滚动卡顿 | scroll-view + backdrop-filter GPU 开销 | 改为原生滚动 + 移除 blur |
| 头像跨用户不显示 | cloud:// fileID 不可跨用户读取 | 云函数转换为 https 临时链接 |
| 管理员下架无权限校验 | banProduct/unbanProduct 缺少 isSuperAdmin 检查 | 补充权限校验 |
| 管理员用户查询失败 | users 集合用 _openid 查询（应为 openid） | 修正字段名 |
| 管理员查看用户详情 404 | 导航到不存在的 admin-user 页面 | 改为 user-profile 页面 |
