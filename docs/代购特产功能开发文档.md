# 代购特产功能开发文档

> 版本: v1.0.0 | 创建日期: 2026-03-27 | 状态: 待开发

---

## 一、功能概述

### 背景

「互换特产」现有核心功能为物物交换，用户须持有自己的特产才能参与。代购特产功能在此基础上新增"直接购买"通道：发布者勾选「可代购」后，任何用户无需拥有特产，直接付款即可购买。

### 角色定义

| 角色 | 说明 |
|------|------|
| **代购者（卖家）** | 在发布特产时勾选「可代购」，填写代购价格和库存 |
| **买家** | 看到带「代购」角标的特产后，直接付款下单购买，无需互换 |
| **平台** | 抽取交易额的 5% 作为服务费（通过微信支付分账实现） |

### 与互换流程的关系

同一件特产可以同时开放互换和代购，两种方式互不影响，但库存为 0 时自动关闭代购。

---

## 二、用户故事

### 代购者视角
1. 发布特产时，勾选「开启代购」复选框
2. 填写代购总价（买家实付）、原价（划线价）、库存数量
3. 发布后，特产在发现页显示橙色「代购」角标
4. 有人下单后，代购者收到通知，在「我的代购订单」中填写快递信息
5. 买家确认收货后，平台将货款（扣除 5% 服务费）结算给代购者

### 买家视角
1. 在发现页看到带「代购」角标的特产
2. 进入详情页，看到代购价格和「立即代购」按钮
3. 填写收货地址，确认订单，完成微信支付
4. 收货后点击「确认收货」，订单完成

## 四、数据库设计

### 4.1 products 集合 — 新增字段

在现有 `products` 集合中新增 `daigou` 嵌套对象：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `daigou.enabled` | Boolean | 是 | 是否开启代购 |
| `daigou.price` | Number | 是* | 买家代购总价（元） |
| `daigou.originalPrice` | Number | 否 | 划线原价（元） |
| `daigou.stock` | Number | 是* | 当前可售库存 |
| `daigou.soldCount` | Number | 是* | 累计销量（初始0） |
| `daigou.serviceFee` | Number | 是* | 平台服务费（5%） |

> `*` 仅当 `enabled = true` 时必填

**示例**：
```json
{
  "daigou": {
    "enabled": true,
    "price": 29.9,
    "originalPrice": 45.0,
    "stock": 12,
    "soldCount": 3,
    "serviceFee": 1.495
  }
}
```

### 4.2 daigouOrders 集合（新建）

独立于 `swapOrders`，专门存储代购订单。

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `_id` | String | 自动 | 文档 ID |
| `orderNo` | String | 是 | 订单号（时间戳+随机6位） |
| `productId` | String | 是 | 特产 ID |
| `productName` | String | 是 | 特产名称（冗余存储） |
| `productImages` | Array | 是 | 特产图片（冗余存储） |
| `sellerId` | String | 是 | 卖家 openid |
| `sellerName` | String | 是 | 卖家昵称（冗余） |
| `buyerId` | String | 是 | 买家 openid（`_openid`） |
| `buyerName` | String | 是 | 买家昵称（冗余） |
| `address` | Object | 是 | 收货地址（见下方结构） |
| `price` | Number | 是 | 成交价（买家实付） |
| `platformFee` | Number | 是 | 平台服务费（5%） |
| `sellerIncome` | Number | 是 | 卖家到账金额（price-platformFee） |
| `status` | String | 是 | 订单状态（见状态机） |
| `expressNo` | String | 否 | 快递运单号 |
| `expressCompany` | String | 否 | 快递公司 |
| `wxTransactionId` | String | 否 | 微信支付交易 ID |
| `payTime` | Date | 否 | 支付时间 |
| `shipTime` | Date | 否 | 发货时间 |
| `completeTime` | Date | 否 | 完成时间 |
| `cancelTime` | Date | 否 | 取消时间 |
| `createTime` | Date | 是 | 创建时间 |

**address 结构**：
```json
{
  "contactName": "张三",
  "contactPhone": "138xxxxxxxx",
  "province": "上海市",
  "city": "上海市",
  "district": "浦东新区",
  "detailAddress": "XX路XX号"
}
```

**订单状态机**：

```
pending_payment（待支付）
    ↓ 支付成功
pending_shipment（待发货）
    ↓ 卖家填写快递
shipped（已发货）
    ↓ 买家确认 / 7天自动确认
completed（已完成）

pending_payment → cancelled（24h未支付自动取消 / 买家主动取消）
pending_shipment → cancelled（买家申请取消 且 24h内 / 平台介入）
pending_shipment → refunding（买家申请退款）
shipped → refunding（买家申请退款）
refunding → refunded（退款成功）
refunding → shipped（卖家拒绝，退回发货状态）
```

### 5.2 daigouMgr（新建云函数）

> 建议独立为新云函数，不与 productMgr / orderMgr 混合

#### `createOrder` — 创建代购订单

**入参**：
```js
{
  productId: String,   // 特产 ID
  addressId: String    // 用户地址 ID（从 addresses 集合读取）
}
```

**返回**：
```js
{
  success: Boolean,
  orderId: String,     // 新建订单 ID
  orderNo: String,     // 订单号
  payParams: Object    // 微信支付参数（timeStamp/nonceStr/package/signType/paySign）
}
```

**业务逻辑**：
1. 检查特产是否存在且 `daigou.enabled = true`
2. 检查库存 ≥ 1
3. 检查买家不能是卖家自己
4. 创建 daigouOrders 文档，状态为 `pending_payment`
5. 调用微信支付统一下单接口，返回支付参数
6. 减库存（乐观锁，防超卖）：`db.command.inc(-1)`

---

#### `getOrderDetail` — 获取订单详情

**入参**：`{ orderId: String }`

**返回**：`{ success, order }`（订单完整信息）

**权限**：仅买家或卖家本人可查

---

#### `getOrderList` — 获取订单列表

**入参**：
```js
{
  role: 'buyer' | 'seller',   // 我买的 / 我代购的
  status: String | null,       // 状态筛选，null = 全部
  page: Number,
  pageSize: Number
}
```

**返回**：`{ success, list, total }`

---

#### `shipOrder` — 卖家发货

**入参**：
```js
{
  orderId: String,
  expressCompany: String,  // 快递公司名称
  expressNo: String        // 运单号
}
```

**返回**：`{ success }`

**业务逻辑**：
- 验证当前用户为订单卖家
- 订单状态必须为 `pending_shipment`
- 更新状态为 `shipped`，记录 `shipTime`

---

#### `confirmReceived` — 买家确认收货

**入参**：`{ orderId: String }`

**返回**：`{ success }`

**业务逻辑**：
- 验证当前用户为订单买家
- 订单状态必须为 `shipped`
- 更新状态为 `completed`，记录 `completeTime`
- 触发分账：将 `sellerIncome` 转入卖家账户（微信支付分账 API）

---

#### `cancelOrder` — 取消订单

**入参**：`{ orderId: String, reason: String }`

**返回**：`{ success }`

**业务逻辑**：
- `pending_payment`：直接取消，库存+1
- `pending_shipment`：仅 24h 内可取消，触发退款，库存+1
- 其他状态：不可取消

---

#### `applyRefund` — 申请退款

**入参**：`{ orderId: String, reason: String }`

**返回**：`{ success }`

**业务逻辑**：
- 状态 `pending_shipment` 或 `shipped` 可申请
- 更新状态为 `refunding`
- 通知卖家处理

---

#### `handleRefund` — 卖家处理退款申请

**入参**：`{ orderId: String, approve: Boolean }`

**返回**：`{ success }`

**业务逻辑**：
- `approve = true`：触发微信支付退款，状态改为 `refunded`，库存+1
- `approve = false`：状态回退为 `shipped`，通知买家

---

#### `paymentCallback` — 微信支付回调（HTTP触发器）

> 由微信支付平台调用，验签后更新订单状态为 `pending_shipment`

## 七、业务规则

### 7.1 价格与费用
- 代购价：¥1.00 ~ ¥9999.00
- 平台服务费：成交价 × 5%（向下取整到分）
- 卖家到账：成交价 × 95%
- 运费：默认由卖家包含在代购价中（不单独展示运费）

### 7.2 超时机制（由 dailyTasks 云函数处理）

| 场景 | 超时时长 | 触发动作 |
|------|---------|---------|
| 待支付 → 自动取消 | 24 小时 | 取消订单，库存+1 |
| 待发货 → 超时提醒 | 48 小时 | 推送提醒卖家 |
| 已发货 → 自动确认收货 | 7 天 | 状态改为 completed，触发分账 |

### 7.3 防超卖
- 下单时使用数据库原子操作（`db.command.inc(-1)`）减库存
- 同时校验 `stock > 0`，若失败则返回「已抢完」
- 取消订单、退款成功后原子性恢复库存

### 7.4 退款规则
- `pending_payment`：直接取消，无需退款（未扣款）
- `pending_shipment`（24h内）：买家可主动申请，自动退款
- `pending_shipment`（超24h）、`shipped`：需申请，卖家同意后退款
- `completed`：不支持退款（走纠纷流程）

## 八、微信支付接入要点

### 8.1 前置条件
- 需绑定微信支付商户号（mchId）
- 需申请分账权限（微信支付后台）
- 卖家需完成实名认证并绑定分账接收方

### 8.2 支付流程
```
前端发起下单 → daigouMgr.createOrder
    ↓
云函数调用微信支付统一下单 API
    ↓
返回 payParams 给前端
    ↓
前端调用 wx.requestPayment(payParams)
    ↓
微信回调 paymentCallback（HTTP触发器）
    ↓
验签 → 更新订单状态为 pending_shipment
```

### 8.3 分账流程
```
买家确认收货 / 7天自动确认
    ↓
daigouMgr.confirmReceived
    ↓
调用微信支付分账 API
    ↓
95% 转入卖家分账账户
5% 留在平台账户
```

## 十、注意事项

1. **数据兼容**：现有 `products` 集合没有 `daigou` 字段，前端读取时需加 `?.` 判空，避免旧数据报错
2. **图片冗余**：订单创建时需将商品图片 fileID 转为 https 临时链接存储到订单中，避免商品下架后订单图片失效
3. **状态隔离**：代购订单与互换订单完全独立，分别存储在 `daigouOrders` 和 `swapOrders` 集合中，互不影响
4. **并发控制**：减库存必须使用原子操作，不能先读后写
5. **支付安全**：支付回调需校验微信签名，防止伪造回调
6. **分账延迟**：微信支付分账要求距支付完成 24h 后方可分账，需在自动确认收货时考虑此延迟
