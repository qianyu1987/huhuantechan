## 一、功能概述

邀请裂变是用户增长的核心功能，通过积分奖励激励用户邀请好友注册并完成首次互换。

### 核心目标
- **低成本获客** - 利用现有用户社交网络拉新
- **提升留存** - 被邀请用户有更高留存率
- **促进活跃** - 奖励与首次互换绑定，确保新用户有真实行为

### 奖励规则
| 事件 | 邀请人奖励 | 被邀请人奖励 |
|------|-----------|-------------|
| 好友注册 | +10 积分 | +5 积分 |
| 好友首次互换完成 | +20 积分 | +10 积分 |

## 三、数据结构设计

### 3.1 users 表（用户）

```javascript
{
  _id: String,
  openid: String,
  nickName: String,
  avatarUrl: String,
  
  // 邀请相关字段
  inviteCode: String,        // 我的邀请码（如 INVABC12）
  invitedBy: String,         // 被谁邀请（邀请人的 inviteCode）
  inviterOpenid: String,     // 邀请人的 openid（用于反查）
  inviteTime: Date,          // 被邀请的时间
  
  // 奖励状态
  signupRewardSent: Boolean, // 注册奖励是否已发放
  firstSwapRewardSent: Boolean, // 首次互换奖励是否已发放
  
  // 统计
  inviteCount: Number,       // 邀请人数
  points: Number,            // 当前积分
  swapCount: Number,         // 互换次数
}
```

### 3.2 invite_rewards 表（邀请奖励记录）- 新建

```javascript
{
  _id: String,
  inviterOpenid: String,     // 邀请人 openid
  invitedOpenid: String,     // 被邀请人 openid
  type: String,              // 'signup' | 'first_swap'
  amount: Number,            // 奖励积分数量
  createTime: Date,          // 发放时间

```

### 3.3 points_log 表（积分记录）

```javascript
{
  _id: String,
  _openid: String,

  amount: Number,            // 积分变动（正数为获得）
  desc: String,              // 描述
  relatedOpenid: String,     // 关联用户 openid（被邀请人/邀请人）
  createTime: Date,

## 四、核心流程设计

### 4.1 邀请流程

```
用户A点击"邀请好友"
    ↓
生成邀请码 + 小程序码
    ↓
分享给好友B（带 inviteCode 参数）
    ↓
好友B打开小程序 → 检测 inviteCode 参数
    ↓
好友B注册成功 → 绑定邀请关系
    ↓
双方获得注册奖励积分
    ↓
好友B完成首次互换 → 双方获得额外积分
```

### 4.2 奖励发放时机

#### 注册奖励（已实现）
```
bindInvite 云函数
    ↓
验证邀请码有效
    ↓
更新被邀请人的 invitedBy 字段
    ↓
邀请人积分 +10
被邀请人积分 +5
    ↓
记录到 points_log
```

#### 首次互换奖励（待实现）
```
orderMgr 云函数 - 订单完成
    ↓
检查是否是首次互换（swapCount === 1）
    ↓
检查是否有邀请人（invitedBy 存在）
    ↓
检查是否已发放首次互换奖励（firstSwapRewardSent === false）
    ↓
邀请人积分 +20
被邀请人积分 +10
    ↓
更新 firstSwapRewardSent = true
    ↓
记录到 points_log 和 invite_rewards
```

// 检查该设备是否已注册过其他账号
const sameDeviceUsers = await db.collection('users').where({
  deviceFingerprint: deviceFingerprint

if (sameDeviceUsers.total > 0) {
  // 标记为可疑，不发奖励或降低奖励
}
```

### 7.2 邀请频率限制

```javascript
// 同一用户每天最多邀请 10 人获得奖励
const todayInvites = await db.collection('invite_rewards').where({
  inviterOpenid: openid,
  type: 'signup',
  createTime: _.gte(todayStart)
}).count()

if (todayInvites.total >= 10) {
  // 超过限制，不发放奖励
}
```

## 九、测试用例

### 9.1 注册奖励测试

1. 用户A生成邀请链接
2. 用户B通过链接打开小程序
3. 用户B完成注册
4. 验证：用户A积分 +10，用户B积分 +5

### 9.2 首次互换奖励测试

1. 用户B（被邀请人）发起互换
2. 订单完成
3. 验证：用户A积分 +20，用户B积分 +10
4. 用户B再次互换，不再发放首次互换奖励

### 9.3 防作弊测试

1. 同一设备注册多个账号
2. 验证：后续账号不发放邀请奖励
3. 同一用户短时间大量邀请
4. 验证：超过限制后不发放奖励
