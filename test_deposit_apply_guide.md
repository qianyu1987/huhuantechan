## 技术实现验证

### 1. 云函数错误处理验证

**`daigouMgr/applyDeposit` 错误处理**：
```javascript
try {
  const addRes = await db.collection('daigouDepositApply').add({ data: applyData })
} catch (dbErr) {
  console.error('[applyDeposit] db.add error:', dbErr)
  // 集合不存在(-502005)或权限问题，给出友好提示
  if (dbErr.errCode === -502005 || (dbErr.message && dbErr.message.includes('collection not exist'))) {
    return { success: false, message: '数据库集合尚未初始化，请联系管理员创建 daigouDepositApply 集合后重试' }
  }
  return { success: false, message: '提交失败：' + (dbErr.message || '数据库写入异常，请重试') }
}
```

## 常见问题排查

### Q1: 点击押金余额条无反应
**可能原因**：
1. `bindtap` 事件未绑定
2. 页面路由未注册
3. 跳转函数错误

**解决方案**：
1. 检查 `mine/index.wxml` 中的 `bindtap`
2. 检查 `app.json` 中的页面注册
3. 检查跳转函数实现

### Q2: 提交申请时报"集合不存在"
**可能原因**：
1. `daigouDepositApply` 集合未创建
2. 集合权限设置错误
3. 云环境错误

**解决方案**：
1. 在云开发控制台创建集合
2. 检查集合权限设置
3. 确认云环境ID正确

### Q3: 返回按钮无效
**可能原因**：
1. `switchTab` 只能用于 tabBar 页面
2. 页面栈深度问题
3. 跳转URL错误

**解决方案**：
1. 使用 `reLaunch` 代替 `switchTab`
2. 检查页面栈状态
3. 确认跳转URL正确

### Q4: 表单验证不通过
**可能原因**：
1. 验证逻辑错误
2. 输入格式不符合要求
3. 必填项未填写

**解决方案**：
1. 检查验证逻辑代码
2. 提供明确的错误提示
3. 高亮显示必填项
