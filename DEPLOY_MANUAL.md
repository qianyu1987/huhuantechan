## 3. 云函数部署

### 3.1 部署方法（推荐：微信开发者工具）

```
1. 打开微信开发者工具
2. 展开左侧 cloudfunctions 目录
3. 右键点击目标云函数目录
4. 选择【上传并部署：云端安装依赖】
5. 等待成功提示
```

> ⚠️ 注意：选【云端安装依赖】而非【本地安装依赖后上传】，避免 node_modules 体积过大。

### 3.2 必须部署的云函数清单

按以下**顺序**部署：

#### 第一批（基础依赖，优先部署）

| 云函数 | 描述 | 关键变更 |
|--------|------|----------|
| `userInit` | 用户初始化、手机号验证 | 修复新用户登录 `actualEvent` 解析、UnionID 跨平台 |
| `adminMgr` | 管理员功能、功能开关 | 引入内存缓存、超管验证缓存10分钟 |

#### 第二批（核心业务）

| 云函数 | 描述 | 关键变更 |
|--------|------|----------|
| `productMgr` | 特产管理 | 图片临时链接缓存1.5h、白名单状态过滤 |
| `orderMgr` | 订单管理 | 无变更，确保线上版本最新 |
| `reviewMgr` | 评价系统 | 无变更，确保线上版本最新 |

#### 第三批（辅助功能）

| 云函数 | 描述 | 关键变更 |
|--------|------|----------|
| `dailyTasks` | 签到、积分、定时任务 | 新功能，首次部署 |
| `resetData` | 测试数据重置 | 仅开发/测试环境，**生产禁止** |

### 3.3 部署后验证

每个云函数部署完成后，在**云开发控制台 → 云函数 → 日志**中确认：
1. 无部署报错
2. 冷启动日志 `[App] 云开发初始化成功` 正常输出


## 7. 常见问题 & 回滚

### Q1: 部署后云函数调用失败 `-501000 resource not found`

**原因**：云函数未成功部署到对应环境  
**解决**：
1. 检查云函数是否部署到 `cloud1-3g4sjhqr5e28e54e`
2. 重新执行【上传并部署：云端安装依赖】
3. 在云开发控制台确认云函数列表中有对应函数

### Q2: 安卓端无数据，提示"网络错误"

**原因**：`project.config.json` 中 `urlCheck: true`  
**解决**：改为 `urlCheck: false` 后重新上传

### Q3: 新用户保存资料失败"无此用户"

**原因**：`userInit` 云函数中使用了 `event` 而非 `actualEvent`  
**解决**：确认已部署最新 `userInit` 云函数（2026-03-26 修复版本）

### Q4: 图片显示 403 错误

**原因**：临时链接已过期（有效期2小时）  
**解决**：
- 云端：`productMgr` 已内置1.5小时缓存，2小时前会重新请求
- 前端：`imageOptimizer` 已内置 LRU 缓存，会自动刷新

### Q5: HarmonyOS 端 -601002 错误

**原因**：云开发环境初始化失败或网络抖动  
**解决**：已内置指数退避重试（1s/2s/5s），通常自动恢复

### 回滚步骤

```
1. 微信公众平台 → 版本管理 → 找到上一个版本 → 发布（线上版本回滚）
2. 云函数回滚：在云开发控制台 → 云函数 → 对应函数 → 版本管理 → 选择上一版本发布
```
