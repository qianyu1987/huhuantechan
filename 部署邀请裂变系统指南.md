## 已完成的修改

### 1. 云函数修改

#### userInit 云函数
**修改内容**:
1. **奖励类型统一**: 将现金奖励改为积分奖励
   - 邀请人注册奖励: 10积分（原0.3元）
   - 被邀请人注册奖励: 5积分（原0.1元）
2. **防作弊机制**:
   - 每日邀请次数限制: 10次/天
   - 防止重复绑定邀请关系
   - 防止自己邀请自己
3. **邀请奖励记录**:
   - 新增 `invite_rewards` 集合记录
   - 记录邀请人和被邀请人关系
4. **集合自动创建**:
   - 自动创建 `invite_rewards` 集合

**修改文件**: `cloudfunctions/userInit/index.js`

#### orderMgr 云函数
**修改内容**:
1. **邀请奖励记录**:
   - 首次互换奖励记录到 `invite_rewards` 集合
   - 保持20积分奖励不变

**修改文件**: `cloudfunctions/orderMgr/index.js`

### 2. 数据库集合

#### 需要创建的集合
1. **invite_rewards** (邀请奖励记录)
   ```javascript
   {
     _id: String,
     inviterOpenid: String,     // 邀请人 openid
     invitedOpenid: String,     // 被邀请人 openid
     type: String,              // 'signup' | 'first_swap'
     amount: Number,            // 奖励积分数量
     orderId: String,           // 订单ID（仅first_swap类型）
     createTime: Date,          // 发放时间
   }
   ```

#### 系统配置
需要在 `system_config` 集合中配置奖励金额：
```javascript
// 邀请奖励配置
{
  configKey: 'invite_reward_inviter',
  configValue: '10',  // 邀请人获得积分
  desc: '邀请好友注册奖励积分'
},
{
  configKey: 'invite_reward_invitee',
  configValue: '5',   // 被邀请人获得积分
  desc: '被邀请注册奖励积分'
}
```

1. **邀请码生成测试**

2. **邀请关系绑定测试**
   - 使用测试账号A生成邀请链接
   - 使用测试账号B通过链接注册
   - 检查邀请关系是否绑定
   - 检查积分奖励是否发放

3. **首次互换奖励测试**
   - 被邀请人B完成首次互换
   - 检查双方是否获得20积分奖励
   - 检查 `invite_rewards` 集合记录

4. **防作弊测试**
   - 测试每日邀请次数限制
   - 测试重复绑定防止
   - 测试自己邀请自己防止

## 故障排除

### 常见问题
1. **邀请码不显示**
   - 检查 `userInit.getInviteData` 云函数
   - 检查用户是否有 `openid`

2. **奖励未发放**
   - 检查 `points_log` 集合
   - 检查 `invite_rewards` 集合
   - 检查云函数日志

3. **防作弊机制误报**
   - 检查时间计算逻辑
   - 检查查询条件

4. **集合不存在错误**
   - 运行云函数触发自动创建
   - 手动创建集合

### 应急回滚
如果部署后出现问题，可以：
1. 恢复之前的云函数版本
2. 临时关闭邀请奖励功能
3. 手动修复数据
