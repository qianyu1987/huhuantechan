# Android 签名证书配置说明

## 使用微信开发者工具生成证书(推荐)

### 步骤一:打开证书管理工具

1. 打开微信开发者工具
2. 打开本项目 `specialty-swap`
3. 点击顶部工具栏的 **"选择设备"** 下拉菜单
4. 选择 **"签名证书管理"** → **"Android 签名证书生成"**

### 步骤二:填写证书信息

在弹出的对话框中填写以下信息:

| 配置项 | 建议值 | 说明 |
|--------|--------|------|
| **证书别名 (Alias)** | `specialty_swap` | 密钥对的标识名称 |
| **证书密码 (Key Password)** | `specialty2026swap` | 保护别名对应密钥的密码 |
| **证书文件密码 (Store Password)** | `specialty2026swap` | 保护整个证书文件的密码 |

**重要提示**: 官方建议证书密码与证书文件密码保持一致,避免潜在问题。

### 步骤三:保存证书文件

生成成功后,微信开发者工具会提示保存路径。建议保存到:

```
c:\Users\31849\WorkBuddy\20260320155547\specialty-swap\android-keystore\specialty-swap.keystore
```

### 步骤四:记录证书信息

生成完成后,请妥善记录以下四个关键信息:

```
证书别名: specialty_swap
证书密码: specialty2026swap
证书文件密码: specialty2026swap
证书文件路径: c:\Users\31849\WorkBuddy\20260320155547\specialty-swap\android-keystore\specialty-swap.keystore
```

## 使用场景

### 1. 真机调试

首次使用"真机调试"或"模拟器调试"时,会弹出签名配置窗口,填入上述四个信息即可。

### 2. 打包发布

在"发行"菜单下选择"打包生成APK/AAB"时,需要**重新填写**一遍证书信息。

**重要**: 调试和发布必须使用同一份证书,否则会导致签名冲突。

### 3. 微信开放平台配置

如需集成微信登录、分享、支付等功能:

1. 使用微信开发者工具的 **"生成 Android 签名与公钥"** 功能
2. 获取应用签名(MD5/SHA值)
3. 在[微信开放平台](https://open.weixin.qq.com)填写应用签名

## 安全提示

⚠️ **证书文件和密码是最高机密,请务必:**

- ✅ 妥善备份证书文件(.keystore)
- ✅ 安全保存所有密码信息
- ✅ 不要上传到 Git 仓库
- ❌ 一旦丢失无法恢复,将无法更新已上架的应用
