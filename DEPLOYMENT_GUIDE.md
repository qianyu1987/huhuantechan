## 待部署的云函数列表

根据最新的修改，以下云函数需要重新部署：

### 1. userInit（最高优先级）
**修改内容**：
- UnionID跨平台用户识别
- 手机号验证功能（阿里云短信API）
- 头像上传修复（cloud://链接处理）

**部署步骤**：
1. 在微信开发者工具中，右键点击 `cloudfunctions/userInit` 文件夹
2. 选择 "上传并部署: 云端安装依赖"
3. 等待部署完成（可能需要1-2分钟）
4. 查看控制台日志，确认显示 "[SMS] 阿里云短信服务初始化成功"

**验证方法**：
- 在小程序端登录，查看云函数日志应显示 `[userInit] 平台信息: { openid, unionid, ... }`
- 数据库 `users` 集合应包含 `unionid` 字段
- 测试手机号验证功能

## 批量部署方法

如果需要一次性部署所有云函数：

1. 在微信开发者工具中，右键点击 `cloudfunctions` 文件夹
2. 选择 "同步云函数列表"
3. 然后依次对每个云函数执行部署操作

## 注意事项

1. **部署顺序**：建议先部署 `userInit`，因为它包含最关键的用户系统修复
2. **网络环境**：确保网络稳定，部署过程中不要关闭开发者工具
3. **依赖安装**：如果部署失败，检查 `package.json` 是否正确
4. **云环境**：确保云开发环境已开通且状态正常
5. **权限**：确保有云开发的部署权限

---

## 部署时间预估

- 单个云函数：1-2 分钟
- 全部云函数：约 10-15 分钟

---

## 遇到问题？

1. **部署失败**：检查网络连接，重试部署
2. **依赖安装失败**：检查 `package.json` 格式，手动在云函数目录执行 `npm install`
3. **云函数调用失败**：检查云开发环境配置，查看云函数日志
4. **手机号验证失败**：检查阿里云短信配置，确认 AccessKey 有效

---

## 部署完成后的操作

1. **重启小程序**：关闭小程序后重新打开
2. **清除缓存**：在开发者工具中点击 "清除缓存"
3. **测试功能**：逐一测试各功能是否正常
4. **查看日志**：监控云函数调用日志，确认无报错
