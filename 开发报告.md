### 2. 云函数 orderMgr 修复

**文件**：`cloudfunctions/orderMgr/index.js`

#### 修复问题：
1. **语法错误**：第618-619行重复声明 `let myAddress`，已删除重复
2. **空数组查询**：修复 `_.in([])` 查询导致的错误，添加空数据保护
3. **用户信息获取**：完善发起方和接收方用户信息查询

#### 关键代码逻辑：
```javascript
// 获取用户信息（订单为空时跳过）
let userMap = {}
if (uniqueOpenids.length > 0) {
  const usersRes = await db.collection('users')
    .where({ openid: _.in(uniqueOpenids) })
    .field({ openid: true, nickName: true, avatarUrl: true, creditScore: true, rating: true, ratingCount: true })
    .get()
  usersRes.data.forEach(u => { userMap[u.openid] = u })
}
```

### 3. 前端 JS 修复

**文件**：`miniprogram/pages/order/index.js`

- **添加缺失的 `formatOrder` 函数**：用于格式化订单数据，生成状态标签、操作按钮、时间显示等
- 该函数是订单列表渲染的关键，缺失会导致页面数据为空
